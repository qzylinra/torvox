## 三、实现详情

### 3.1 torvox-core — 核心类型系统 (no_std)

**设计原则**: 零依赖, 零 unsafe, no_std + alloc

| 类型 | 文件 | 用途 | 关键特性 |
|------|------|------|----------|
| `Cell` | cell.rs | 单个终端字符 | char + fg/bg Color + Attrs |
| `Color` | cell.rs | RGBA 颜色 | new(), from_ansi(256色), serde |
| `Attrs` | cell.rs | 10 个 SGR 属性 | bold/dim/italic/underline/double_underline/reverse/strikethrough/blink/hidden/overline |
| `DirtyMask` | cell.rs | 脏行位标志 | Vec\<u64\> 分区, 每 u64 覆盖 64 行, 支持任意行数 |
| `Grid` | grid.rs | 终端网格 | Vec\<Line\>, get/mark/clean/resize/scroll/insert/delete |
| `Line` | line.rs | 单行 | Vec\<Cell\>, get/resize |
| `TerminalConfig` | config.rs | 终端配置 | rows/cols/scrollback_lines/shell |
| `Shell` | config.rs | Shell 类型 | SystemDefault/Custom(String) |
| `CursorState` | cursor.rs | 光标状态 | row/col/style/visible, 移动方法 |
| `Selection` | selection.rs | 文本选择 | Char/Word/Line/Block 模式 |
| `TerminalEvent` | event.rs | 事件枚举 | 9 个变体 (含 DirtyRegion) |
| `SgrAttribute` | ansi.rs | SGR 属性 | 枚举定义，256色查找表 |
| `UnicodeWidth` | unicode.rs | 字符宽度 | 手写宽度表，CJK/Hangul/Fullwidth |

**no_std 兼容性**: `extern crate alloc` 用于 Vec/String，`serde` 和 `postcard` 使用 `default-features = false` + `alloc` feature。`thiserror` 设为 optional，仅 `std` feature 启用。

### 3.2 torvox-terminal — 终端仿真引擎

#### PTY (pty.rs)
- **接口**: `PtyPair::spawn(shell, rows, cols)` — openpty + fork + setsid + login_tty
- **Raw Mode**: termios 配置 — 关闭 echo/canonical/ISIG/IXON, 设置 CS8/VMIN=1/VTIME=0
- **非阻塞**: `set_nonblocking()` — fcntl O_NONBLOCK
- **Kill**: Drop 时递增式终止 — SIGHUP → SIGCONT → 100ms → SIGKILL → waitpid
- **unsafe**: 仅 fork() 块，注释安全不变量

#### VT 解析器 (terminal.rs + parser.rs)
- **vte 0.15**: Paul Williams 状态机, `vte::Perform` trait 实现
- **CSI 序列**: CUU/CUD/CUF/CUB/CUP/CHA, ED/EL/ICH/DCH/ECH, IL/DL/SU/SD, SGR (0-107 + 256色 + 真彩色), DECSET/DECRST, 滚动区域, 光标保存/恢复, DECSCUSR
- **ESC 序列**: 保存/恢复 (7/8), RIS (c), HTS (H), RI (M), 字符集定义 ((/))
- **OSC 序列**: 窗口标题 (0/2)
- **状态**: Grid + CursorState + saved_cursor + scroll_region + tab_stops + modes(BTreeSet) + charsets[4] + alt_grid + title

#### 会话编排 (session.rs)
- **Session**: 拥有 PtyPair + TerminalState + VtParser
- **读取线程**: dup'd fd → non-blocking read → crossbeam channel → 解析
- **写入**: PtyPair.write() 直通
- **等待线程**: waitpid() 检测子进程退出

#### 输入引擎 (keyboard.rs)
- **Kitty 协议**: CSI u 编码, 修饰键位掩码
- **VT 传统**: Esc/CSI/SS3 前缀, 功能键 F1-F20
- **鼠标 SGR**: \e[<button;col;row;M/m 编码
- **括号粘贴**: \e[200~ ... \e[201~

### 3.3 torvox-renderer — GPU 渲染管线

#### 字体管线 (font.rs)
- **FontPipeline**: fontdb 字体发现 → cosmic-text FontSystem → swash 缩放/光栅化 → etagere 图集打包
- **字形缓存**: HashMap\<GlyphKey, GlyphInfo\>
- **图集**: 2048×2048 RGBA 位图, shelf packing
- **预光栅化**: ASCII 32-126

#### GPU 管线 (gpu.rs)
- **GpuContext**: wgpu v29 Instance/Device/Queue/Surface
- **着色器**: cell.wgsl (实例化四边形, atlas UV), cursor.wgsl (纯色矩形)
- **CellInstance**: 72 字节 (pos, atlas UV, fg/bg colors, flags)
- **渲染流程**: 绑定管线 → 设置 bind group → 创建实例缓冲区 → draw(0..6, 0..N)
- **正交投影**: 屏幕坐标变换

#### 空壳 (pipeline.rs)
- RenderPipeline 仅有空结构体和 Default impl

### 3.4 torvox-gui-android — Android 桥接

#### UniFFI 桥接 (bridge.rs)
- **BridgeCell**: char_code(u32) + fg/bg(u32) + BridgeAttrs
- **Shell**: SystemDefault/Custom(path) 枚举
- **TerminalConfig**: shell/rows/cols/scrollback_lines
- **TerminalEvent**: 6 变体 (Bell/TitleChanged/ClipboardRequest/HyperlinkHover/ProcessExited/DirtyRegion)
- **TerminalError**: PtyError/InvalidConfig (使用 `detail` 而非 `message`)
- **TorvoxBridge**: ping/spawn_terminal/get_config/echo_cells
- **From/Into**: 全部 core↔bridge 类型双向转换

### 3.5 torvox-exec — W^X 多调用二进制

- 读取 argv[0] 的 file_name() 确定身份
- 直接模式: torvox-exec \<command\> [args...]
- 符号链接模式: ln -s torvox-exec ls; ./ls [args...]

## 六、潜在问题与风险

### 6.1 技术风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| wgpu Android Surface 集成复杂 | P1.5 可能延期 | 参考 Alacritty/winit Android 支持 |

### 6.2 架构风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| renderer→terminal 反向依赖 | 无法独立测试 renderer | P1.5 前重构: 将 build_cell_instances 移到 gui-android 或引入 GridSnapshot trait |
| 无事件通知机制 | 渲染线程轮询浪费 CPU | 集成 crossbeam::Notify 或 Condvar |
| Scrollback 未集成 | 回滚功能完全不可用 | P2.1 前修复 Grid scrollback |
| FFI 边界传递结构化事件 | TerminalEvent 桥接不完整 | 扩展 bridge.rs TerminalEvent 变体 |

### 6.3 代码质量风险

| 风险 | 影响 | 缓解 |
|------|------|------|
| build_cell_instances 每帧重建 FontPipeline | 帧率灾难性下降 | 传递 &mut FontPipeline 参数 |
| VT 解析器无模糊测试 | 边缘输入可能 panic | P1.1 后添加 cargo-fuzz |
| 无 MIRI 验证 | 可能有未定义行为 | CI nightly 添加 MIRI |
| proptest 无实际策略 | 属性测试形同虚设 | 实现 proptest 策略 |

## 七、注意事项

### 7.1 开发流程

1. **必须先读 ROADMAP** — 确认当前阶段可达再动手
2. **必须先读相关 ADR** — 修改 PTY 读 ADR 004，修改渲染读 ADR 003
3. **类型先行** — 先定义类型再实现行为
4. **小步提交** — 每步 cargo clippy + cargo test
5. **同步 bridge.rs** — 修改 core 公共类型时必须检查桥接
6. **不引入新依赖** — 除非 ARCHITECTURE.md 版本锁定表已更新

### 7.2 已知陷阱

| 陷阱 | 教训 |
|------|------|
| UniFFI setup_scaffolding!() 只能有一个 | 跨 crate derive 导致重复脚手架 |
| UniFFI Error message 字段与 Kotlin 冲突 | 使用 detail 而非 message |
| DirtyMask 不再是 Copy | Vec\<u64\> 含堆分配 |
| Shell::Custom(String) 不是 Copy | TerminalConfig 也失去 Copy |
| thiserror 需要 std feature | no_std 中设为 optional |
| cargo-ndk v4 CLI 重写 | 与 v3 脚本不兼容 |
| nix 0.31 API 变更 | fcntl 接受 AsFd 而非 RawFd |

### 7.3 构建验证命令

```bash
# 每次修改后必须运行
cargo clippy -- -D warnings
cargo test --workspace
cargo fmt --check

# 阶段完成后
./scripts/quality-gate.sh

# no_std 验证
cargo build -p torvox-core --target thumbv6m-none-eabi --no-default-features --features alloc
```
